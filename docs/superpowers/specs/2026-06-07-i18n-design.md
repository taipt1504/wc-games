# Full i18n (EN/VI) + Brand Rename — Design

**Date:** 2026-06-07
**Status:** Approved (brainstorming) → ready for implementation plan

## Goal

1. Rename the brand **GOLAZO → "World Cup Games"** everywhere it's user-visible.
2. Add **full English + Vietnamese i18n** to the player-facing web app: a lightweight locale context, EN/VI catalogs, a language switcher, locale persistence, and extraction of all player-facing UI strings into the catalogs.

## Decisions (locked during brainstorming)

- **Scope:** player-facing screens + app shell + nav + toasts/errors. **Admin console stays English** (internal ops).
- **Mechanism:** lightweight custom React context (no i18n library). Fits the single client-router app (no per-locale URLs, no middleware).
- **Persistence:** `wcg_locale` cookie (so the server layout can set `<html lang>`/title) **+** `localStorage` (instant client read). First visit: detect from `navigator.language` (Vietnamese → `vi`, else `en`). Manual switch overrides + persists. **No DB changes.**
- **Switcher:** compact EN/VI toggle in the authed top bar **and** the logged-out landing/auth header.
- **Brand:** "World Cup Games" is a constant, **not translated** (same in both locales).

## Background — current state

- Next 15, **client-heavy**: 12 `'use client'` files; `app/layout.tsx` is the only server component (`metadata.title`, `<html lang="en">`). One client router (`app-shell.tsx`).
- ~5k LOC of player components with hardcoded English strings (`screens-core` 505, `screens-match` 613, `screens-compete` 971, `screens-lobby` 945, `screens-tournament` 329, `screens-news` 140, `app-shell` 353, `notification-bell`, `ui` 228). `screens-admin` (1,584) is out of scope.
- No existing i18n/intl dependency.
- GOLAZO appears (user-visible): `app/layout.tsx` title, `screens-core.tsx:207` landing wordmark, `app/api/og/share/route.tsx` OG image, `app/api/v1/news/route.ts` source fallback. Code comments mention GOLAZO — left as-is (not user-visible).

## 1. Locale infrastructure (`apps/web/lib/i18n/`)

- **`locales.ts`**
  ```ts
  export type Locale = 'en' | 'vi';
  export const LOCALES: Locale[] = ['en', 'vi'];
  export const DEFAULT_LOCALE: Locale = 'en';
  export const LOCALE_COOKIE = 'wcg_locale';
  export const BRAND = 'World Cup Games';
  export const LOCALE_LABELS: Record<Locale, string> = { en: 'EN', vi: 'VI' };
  export function normalizeLocale(v: string | undefined | null): Locale {
    return v === 'vi' ? 'vi' : 'en';
  }
  export function detectBrowserLocale(): Locale {
    if (typeof navigator === 'undefined') return DEFAULT_LOCALE;
    return navigator.language?.toLowerCase().startsWith('vi') ? 'vi' : 'en';
  }
  ```

- **`dictionaries/en.ts`** — canonical nested catalog (source of truth for shape). Namespaces: `common, nav, landing, auth, home, schedule, match, betslip, bets, wallet, leaderboard, profile, lobby, tournament, news, toast`. Example shape:
  ```ts
  export const en = {
    common: { save: 'Save', cancel: 'Cancel', back: 'Back', loading: 'Loading…', confirm: 'Confirm' },
    nav: { home: 'Home', schedule: 'Schedule', lobbies: 'Lobbies', tournament: 'Tournament', news: 'News' },
    home: { greeting: 'Welcome back, {name}', pointsLabel: 'Your points' },
    // … one entry per visible string, filled during extraction
  } as const;
  export type Dict = typeof en;
  ```

- **`dictionaries/vi.ts`** — `export const vi: Dict = { … }` (typed `Dict`), so **TS fails the build if any EN key lacks a VI value** (compile-time parity).

- **`dictionaries/index.ts`** — `export const dictionaries: Record<Locale, Dict> = { en, vi };`

## 2. Provider + hook (`apps/web/lib/i18n/provider.tsx`, client)

- `I18nProvider({ initialLocale, children })`:
  - state `locale`, initialized from `initialLocale` (server cookie) — on mount, reconcile with `localStorage`, else `detectBrowserLocale()`, else default; write back so all three agree.
  - `setLocale(l)`: set state + `localStorage[LOCALE_COOKIE]=l` + `document.cookie = '${LOCALE_COOKIE}=${l};path=/;max-age=31536000'` + `document.documentElement.lang = l`.
  - context value `{ locale, setLocale }`.
- **`useT()`** (`hooks.ts`):
  - reads context; **if no provider, falls back to the EN dict** (so standalone-rendered components/tests render English).
  - returns `{ t, locale, setLocale, fmt }`.
  - `t(path: string, vars?: Record<string, string|number>): string` — resolve dotted path against `dictionaries[locale]`; on miss → same path against `en`; on miss → return `path` (and `console.warn` in dev). Interpolate `{var}` tokens from `vars`.
  - `fmt = { date(d, opts?), num(n, opts?) }` — `new Intl.DateTimeFormat(locale, …)` / `Intl.NumberFormat(locale, …)`; replaces bare `toLocaleDateString()` calls.

## 3. Server wiring (`app/layout.tsx`)

- Read the cookie server-side: `const locale = normalizeLocale((await cookies()).get(LOCALE_COOKIE)?.value)`.
- `<html lang={locale}>`.
- `metadata`: title `'World Cup Games — World Cup 2026 Prediction Game'` (static; brand not localized).
- Wrap children: `<I18nProvider initialLocale={locale}>{children}</I18nProvider>` (client provider receiving the server-resolved cookie as the SSR seed → no flash).

## 4. Switcher + brand

- **`LangSwitch.tsx`** (client) — compact segmented EN/VI toggle using `useT().locale`/`setLocale`. Reuses existing chip/segmented styling.
- Placement: `app-shell.tsx` authed top bar (near the notification bell) **and** the public/landing header (`pubbar`) so it's reachable logged-out.
- **Brand rename** → "World Cup Games": `app/layout.tsx` title, `screens-core.tsx` landing wordmark (render `{BRAND}`), `app/api/og/share/route.tsx` wordmark text, `app/api/v1/news/route.ts` `'GOLAZO wire'` → `'World Cup Games wire'`.

## 5. String extraction (phased)

Replace hardcoded player-facing strings with `t('ns.key', vars?)` and dates with `fmt.date(...)`, filling **both** catalogs as each file is done. Order (one file per phase keeps each change reviewable):

- **P0 — infra:** `locales.ts`, `dictionaries/{en,vi,index}.ts` (seeded with `common`/`nav`/`landing` namespaces), `provider.tsx`, `hooks.ts`, `LangSwitch.tsx`, layout server wiring, brand rename, + tests. Wire `LangSwitch` into app-shell + landing.
- **P1** `screens-core.tsx` (landing, auth, home).
- **P2** `app-shell.tsx` + `notification-bell.tsx` + user-facing `ui.tsx` (nav, toasts, shared chrome).
- **P3** `screens-match.tsx` (schedule, match detail, bet slip).
- **P4** `screens-compete.tsx` (bets, wallet, leaderboard, profile).
- **P5** `screens-lobby.tsx` (lobbies, create, view, chat, borrow).
- **P6** `screens-tournament.tsx` + `screens-news.tsx`.

Each phase: extract that file's strings → add keys to `en.ts` + `vi.ts` → file renders via `t()` → `tsc` + tests green → stop for review.

## 6. Conventions

- **Keys** namespaced by screen/feature, lowerCamel leaf names; reuse `common.*` for shared words (Save/Cancel/Back/Confirm).
- **Interpolation** `{name}`, `{count}` — no positional args.
- **Pluralization:** VI has no plural inflection; EN cases are few. Use explicit keys (`bets.oneBet` / `bets.manyBets`) or `{count}`-with-noun where natural. No plural library (YAGNI).
- **Brand** via `BRAND` constant, never a translation key.
- **EN values must byte-match the current strings** so existing tests (which assert English, default locale = EN) keep passing.

## 7. Testing

- **Unit (`hooks.test.ts` / a pure resolver):** `t()` resolves nested path; interpolates `{var}`; missing key → EN fallback → path; `fmt.date`/`num` honor locale.
- **Parity (`dictionaries.test.ts`):** runtime check that `vi` has every leaf key of `en` (belt-and-suspenders beside the `Dict` type) + no empty values.
- **Switcher:** `setLocale('vi')` flips `t()` output for a sample key.
- **Regression:** the 113 existing web tests must stay green — they render with the EN default (no provider → EN fallback), and EN catalog values equal the prior literals.
